## 设计原则

### 命名规范

**标签格式**：采用 `#一级分类/二级分类/三级分类/四级标签` 的四级结构

**层级深度**：
- **一级分类**：实体大类（人物、事件、概念、文献、机构、地点）
- **二级分类**：学科/领域（政治、经济、思想、文化、军事...）
- **三级分类**：细分领域或时期（改革派、先秦、改革开放...）
- **四级标签**：具体属性标签（核心人物、边缘人物、创始人...）

**命名约定**：
- 全部使用中文
- 使用 `/` 分隔层级
- 避免使用数字编号（如「01」），使用语义化名称
- 标签名与文件名解耦，标签用于分类检索

## 实体类型定义

基于现有插件架构，扩展为 6 种核心实体类型：

### 1. 人物（person）

**基础属性**（继承自 Entity）：

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `type` | `'person'` | ✅ | 固定值 |
| `name` | `string` | ✅ | 人物姓名 |
| `time_start` | `Date` | ✅ | 出生日期/生平起始 |
| `time_end` | `Date` | ❌ | 逝世日期/生平结束 |
| `tags` | `string[]` | ✅ | 标签数组 |
| `importance` | `1-5` | ❌ | 重要性等级 |

**人物专属属性**：

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `courtesy_name` | `string` | ❌ | 字 |
| `art_name` | `string` | ❌ | 号 |
| `born_place` | `string` | ❌ | 出生地 |
| `died_place` | `string` | ❌ | 逝世地 |
| `nationality` | `string` | ❌ | 国籍/朝代 |
| `ethnicity` | `string` | ❌ | 民族 |
| `roles` | `string[]` | ❌ | 身份角色（政治家、学者、军人等） |
| `school` | `string` | ❌ | 学派（儒家、法家、自由主义等） |
| `teachers` | `string[]` | ❌ | 师承（人名数组） |
| `students` | `string[]` | ❌ | 弟子（人名数组） |
| `affiliations` | `string[]` | ❌ | 所属组织/机构（机构名数组） |
| `works` | `string[]` | ❌ | 代表作（文献名数组） |
| `core_ideas` | `string[]` | ❌ | 核心思想/主张 |
| `opponents` | `string[]` | ❌ | 论敌/对立人物（人名数组） |
| `era` | `string` | ❌ | 时代/时期 |

## 使用建议

### 基本使用

**根据实际需要选择填写属性，不必填满所有字段**

### 必填字段建议

| 实体类型 | 必填字段 |
|---------|---------|
| **人物** | `name`、`type`、`time_start`、`roles`、`tags` |
| **事件** | `name`、`type`、`time_start`、`location`、`tags` |
| **概念** | `name`、`type`、`category`、`definition`、`tags` |
| **文献** | `name`、`type`、`author`、`work_type`、`tags` |
| **机构** | `name`、`type`、`time_start`、`org_type`、`tags` |
| **地点** | `name`、`type`、`loc_type`、`tags` |

### 最佳实践

1. **`importance` 字段**：影响时间轴上的卡片样式，建议为重要实体设置 4-5
2. **`tags` 数组**：应包含至少一个主分类标签
3. **关联字段**：如 `participants`、`related_events` 等，使用实体名称便于搜索和检索
4. **日期格式**：使用 ISO 8601 格式（YYYY-MM-DD）
5. **标签层级**：根据实际需要选择 2-4 级，不必总是使用 4 级

- `importance` 字段控制卡片样式
- `tags` 字段用于轨道分组和筛选
